# 模块化
  - 模块化主要是用来抽离公共代码，隔离作用域，避免变量冲突
  - IIFE
    - 使用自执行函数来编写模块化
    - 特点：在一个单独的函数作用域中执行代码，避免变量冲突
  - AMD
    - 使用requireJS来编写模块化
    - 特点：依赖必须提前声明好
  - CMD
    - 使用seaJS来编写模块化
    - 特点：支持动态引入依赖文件
  - CommonJS
    - nodejs中自带的模块化
  - UMD
    - 兼容AMD, CommonJS模块化语法
  - Webpack(require.ensure)
    - webpack2.x版本中代码分割
  - ES Module
    - ES6引入的模块化，支持import来引入另一个js
